## جریان داده از طریق سرویس‌ها: REST و RPC

وقتی فرآیندهایی دارید که نیاز به ارتباط از طریق شبکه دارند، چند روش مختلف برای تنظیم آن ارتباط وجود دارد. رایج‌ترین ترتیب داشتن دو نقش است: مشتری‌ها و سرورها. سرورها یک API را از طریق شبکه ارائه می‌دهند، و مشتری‌ها می‌توانند به سرورها متصل شوند تا درخواست‌هایی به آن API ارسال کنند. API ارائه شده توسط سرور به عنوان یک سرویس شناخته می‌شود.

وب به این روش کار می‌کند: مشتری‌ها (مرورگرهای وب) درخواست‌هایی به سرورهای وب ارسال می‌کنند، درخواست‌های GET برای دانلود HTML، CSS، جاوااسکریپت، تصاویر و غیره، و درخواست‌های POST برای ارسال داده به سرور. API شامل مجموعه‌ای استاندارد از پروتکل‌ها و قالب‌های داده (HTTP، URL‌ها، SSL/TLS، HTML و غیره) است. از آنجا که مرورگرهای وب، سرورهای وب و نویسندگان وب‌سایت عمدتاً بر سر این استانداردها توافق دارند، می‌توانید از هر مرورگر وبی برای دسترسی به هر وب‌سایتی استفاده کنید (حداقل در تئوری!).
